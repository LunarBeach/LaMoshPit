LaMoshPit

User Manual & Control Reference
Datamoshing & glitch video encoder — every control explained

§1 — What LaMoshPit Does

LaMoshPit is a datamoshing tool. It takes an H.264 video, lets you manipulate its internal encoder parameters at the macroblock and frame level, then re-encodes a new H.264 file with those manipulations baked in. The result is opened automatically in the Preview Player for inspection.

Every Apply action reads the most recently encoded file and produces a new encoded file. Effects compound across repeated applies — you are always encoding on top of whatever damage the previous pass introduced.

The Golden Rule
Each area of the UI maintains its own independent state. Pressing one area's Apply button does not read or clear the state of other areas. Understanding what each Apply button actually submits to the encoder is the key to using the app effectively.

The Three Independent Control Areas

① Quick Mosh

  • Choose a named preset
  • Press Mosh Now!
  • Whole-video effect, auto-runs encode

② MB Editor

  • Paint macroblocks on frame canvas
  • Dial in per-frame/per-MB effects
  • Press Apply MB Edits

③ Global Encode Params

  • Set encoder-level settings
  • Choose a named preset
  • Press Apply Global Params + Re-encode

§3 — Timeline & Frame Selection

The timeline strip at the top of the window shows every frame of the loaded video as a small coloured block. The colour indicates the frame type detected by the H.264 bitstream analyser:

White — I-frame (IDR keyframe)

Intra-coded. Contains a full picture with no reference to other frames. The encoder's reset point. Datamoshing requires eliminating these (except frame 0) to maintain a continuous chain of inter-predicted frames.

Blue — P-frame

Predicted from a previous frame. Most of the data is motion vectors and residuals relative to the reference. The datamosh smear lives here. Corrupting the reference that P-frames point to produces the smear effect.

Pink — B-frame

Predicted from both a past and a future frame. Typically disabled when Kill I-Frames is active, because B-frame adaptive lookahead can re-insert I-frames unexpectedly.

Selecting Frames

  • Click a frame to select it. The MB editor canvas loads that frame.
  • Click and drag across frames to select a range.
  • The selection label below the timeline shows how many frames are selected.
  • The selection feeds the active range in the MB editor — any knob change you make writes to all selected frames simultaneously.
After a successful encode
The timeline automatically re-analyses the new file and repaints. If Kill I-Frames worked, you should see only a single white block at frame 0 and all blue from there on.

§4 — Frame Operation Buttons

These buttons above the timeline all trigger a full re-encode of the video. They respect the current Global Params panel settings. MB edits are also applied during the re-encode (selected frames get pixel manipulation applied first).

Button What It Does Effect on the Output Video When to Use It
Force → I Re-encodes selected frames as intra-coded keyframes, regardless of what they were originally. Creates hard reset points in the motion chain. The encoder generates a full picture at those positions, wiping any inter-predicted corruption that had accumulated. Rarely useful for datamoshing. Use to punctuate a smear with a sharp clean frame.
Force → P Re-encodes selected frames as P-frames (inter-predicted from the frame before). With Kill I-Frames also checked, all frames — selected or not — are forced to P. Breaks the intra cycle. Frame 0 is always kept as IDR (cannot be P — there is no preceding reference). All other frames become motion-compensated predictions, enabling the inter-prediction chain required for datamoshing. Fundamental datamosh setup. Select all frames → Force P → enables smear/freeze effects from MB edits.
Force → B Re-encodes selected frames as B-frames (bi-predicted from both a past and future frame). Produces temporal ghosting in both directions. The encoder averages content from before and after the frame, creating a translucent double-exposure look. Atmospheric temporal blur. Combine with high ref-frame count for complex ghosting. Disable Kill I-Frames first — B-frames and infinite GOP can conflict.
✖ Delete Physically removes selected frames from the compressed H.264 bitstream using a packet-level splice — no decode or re-encode takes place. The surviving compressed P-frames are copied byte-for-byte to the output. Classic datamosh. The surviving P-frames after the cut still contain their original motion vectors, which reference the content of the deleted frames. The decoder applies those vectors to whatever content is now in its reference buffer (the last frame from before the deleted section). This mismatch — wrong reference content, intact motion vectors — produces the smear, bleed-through, and echo artifacts. Requires an I/P-only stream; use Force → P with Kill I-Frames ON first. The original datamosh technique. First Force → P all frames (with Kill I-Frames ON), then Delete a range of frames at a scene cut. The P-frames after the cut will inherit motion vectors computed against the deleted content, smearing across the wrong reference shot.
◁ Dup Each selected frame is output twice — the copy appears before the original in the stream. Each frame is encoded twice in sequence. With inter-prediction this creates a stuttered freeze: the encoder references the same content twice, producing skip macroblocks and a hold-frame feel. Stutter / freeze effects. Combine with MB edits on the selected frame for a freeze that transitions into corruption.
Dup ▷ Same as Dup Left but the copy appears after the original. Identical to Dup Left but copy order is reversed. When you want the clean version to come first, followed by the repeated/corrupted version.
Frame 0 Cannot Be P or B
The very first frame of any H.264 video must be an IDR keyframe — the decoder has nothing to reference before it. The app shows a warning and leaves frame 0 untouched if you try to Force P/B on it. This is the one I-frame that should remain.

§5 — Quick Mosh Panel

Quick Mosh provides one-click whole-video datamosh effects. Each preset encodes a carefully tuned set of frame-level pixel manipulations across the full clip. Pressing Mosh Now! always forces Kill I-Frames = ON and GOP Size = ∞ — the two non-negotiable prerequisites for datamoshing — regardless of what you have set in the Global Params panel.

Quick Mosh Isolation Rules
  • Ignores whatever is currently in the MB editor
  • Overwrites the MB editor with its own edit map (so you can inspect/modify after)
  • Reads most of the Global Params panel (ME method, QP settings, deblocking, etc.)
  • Forces killIFrames = true and gopSize = ∞ on top of those params

Preset Strategies

Quick Mosh presets use two strategies:

CASCADE (Seed + Propagate)

Seeds one frame (the first P-frame after the IDR) with pixel manipulations, then automatically generates decaying descendant edits for all subsequent frames. The corruption spreads forward through the inter-prediction chain. The decay is linear — at Decay=50%, the effect is at 100% intensity at frame 1 and 50% intensity at the final frame.

STAMP ALL (Per-Frame)

Applies the same manipulation to every single frame independently. No cascade — each frame is stamped with identical pixel changes. Works without a temporal reference chain, so it is effective even on clean source material without needing a long P-frame run.

Preset Reference

Ghost Smear
CASCADE · seed frame 1
Seeds frame 1 with an 80% ghost blend toward the previous frame. The encoder copies content from the preceding frame into each MB, then the cascade propagates this "echo" across the entire clip. Result: a translucent ghost of the opening image drifts over the whole video like a persistent double-exposure.
MV Liquify →
CASCADE · seed frame 1
Seeds with 70px horizontal MV drift × 3 amplify. Replaces each MB's pixels with content from 210 pixels to the right in the reference frame. The encoder sees this content at the wrong position and uses "wrong" motion vectors, causing horizontal smearing throughout. Result: the image appears to melt or flow rightward.
MV Liquify ↓
CASCADE · seed frame 1
Same as MV Liquify → but downward. Content drips downward across the full clip. Result: gravity-melt aesthetic, strong on videos with vertical motion.
Chroma Bleed
CASCADE · seed frame 1
Seeds with independent chroma plane drift (U: +45px, V: −25px) and a DC colour offset. The luma (brightness) channel is left mostly intact while colour planes drift to different positions. Result: VHS-analogue colour bleed where colour separates from luminance, spreading across the clip.
Vortex
CASCADE · seed frame 1
Combines diagonal MV drift (30px right, −30px up) at 4× amplify with 60-unit noise, 20-unit pixel offset, and a 3-MB spill radius. Multiple effects compound in one seed. Result: chaotic spiral/vortex corruption that radiates outward from the seed frame across the whole clip.
Scatter Dissolve
CASCADE · seed frame 1
Seeds with reference scatter (25px radius) and sample blur blending. Each MB samples reference content from random nearby offsets instead of the exact position. Result: a shattered-glass or cubist dissolve effect that decays gradually across the video.
Full Freeze
CASCADE · seed frame 1 · cascadeDecay=0
Seeds with 100% ghost blend and QP=51. Frame 1 becomes a pixel-perfect copy of frame 0. The cascade propagates this at constant intensity with zero decay. Result: the opening frame is frozen for the entire video duration. The encoder outputs skip-MB P-frames with zero residual — the most minimal possible bitstream.
Pixel Disintegrate
STAMP ALL · every frame
Stamps every frame with heavy noise (±160) and a +30 brightness offset, plus QP=45. No reference dependency. Result: the image is progressively erased into high-QP posterised blocks with heavy grain. The encoder cannot resolve fine detail so large flat colour regions dominate.
UV Colour Twist
STAMP ALL · every frame
Stamps every frame with independent U/V DC offsets (+50 U, −50 V) and a 30% ghost blend toward the previous frame. Pushes the chroma planes in opposite directions. Result: hue rotates dramatically across the clip — the image takes on a strong tinted cast that pulses with motion.
Block Melt
STAMP ALL · every frame
Stamps every frame with 80% block flatten (each macroblock's luma is collapsed to its spatial average DC value) and QP=40 plus noise. The encoder receives perfect 16×16 uniform blocks. Result: the image is reduced to coarse mosaic tiles whose colour is the average of the original, with heavy quantisation artefacts at the edges.

§6 — MB Editor (Macroblock Editor)

The MB Editor is the most detailed control surface. It lets you paint specific regions of specific frames and apply a rich set of pixel-level effects to exactly those regions. Pressing Apply MB Edits re-encodes the video applying all those per-frame per-MB manipulations.

Canvas & Painting

  • The canvas shows the current frame decoded from the source video. The 16×16 macroblock grid is overlaid.
  • Click or drag to paint macroblocks. Painted MBs are highlighted.
  • Leave all MBs unpainted = Global Mode. When no MBs are painted on a frame, the effects apply to every macroblock in that frame.
  • Brush size sets how many MBs are painted per stroke.
  • The Clear button removes all painted MBs from the current frame.
  • Navigate frames using the timeline — the canvas updates to show the selected frame.

Active Frame Range

When you select multiple frames in the timeline (click and drag), knob changes write to all selected frames simultaneously. This lets you paint an effect across a whole section without navigating frame by frame.

Cascade Sliders Do NOT Range-Propagate
The Cascade Length and Cascade Decay sliders always write only to the currently-displayed frame, not to the whole selection. This is intentional — setting cascadeLen on every frame in a 300-frame range would turn every frame into a cascade seed, and they would all block each other's descendants.

Knob Reference — MOTION section

Control Range What It Technically Does Visual Result
Ref Depth 0 – 7 Which frame in the reference ring buffer to use as the pixel source for MV drift, ghost blend, chroma drift, and scatter. 0 = no reference (disables all reference-dependent effects). 1 = previous frame. 2 = two frames back, etc. Must be ≥ 1 for MV Drift, Ghost Blend, Chroma Drift, or Ref Scatter to have any effect. Set to 0 and those knobs are silently ignored.
MV Drift X −127 – +127 px Replaces selected MB pixel content with content from the reference frame shifted horizontally by this many pixels (multiplied by MV Amplify). The encoder now sees a frame that looks like it came from a shifted position, so it generates motion vectors pointing there. Horizontal smear / liquify. Positive = content appears to slide left; negative = slides right. Requires Ref Depth ≥ 1.
MV Drift Y −127 – +127 px Same as MV Drift X but vertical. Vertical smear. Positive = slides up; negative = slides down. Requires Ref Depth ≥ 1.
MV Amplify 1 – 8× Multiplies both MV Drift X and MV Drift Y. Effective drift = Drift × Amplify. Amplify=3, DriftX=60 → 180px effective shift. Scales the strength of the MV drift effect. High amplify with moderate drift values produces extreme positional shifts without needing large raw drift numbers.
Ghost Blend % 0 – 100 % Blends selected MB pixels toward the reference frame pixels at the same spatial position (no shift). 100% = completely replace current frame with reference content. 50% = average of current and reference. Temporal echo / ghost trail. The previous frame's content bleeds through into the current frame at the selected opacity. At 100% combined with QP=51, the encoder outputs skip-MBs (zero data) — the reference is propagated verbatim. Requires Ref Depth ≥ 1.
Sample Radius 0 – 8 MB When > 0, the ghost blend samples from a blurred neighbourhood of the reference rather than the exact pixel position. Sample radius in macroblock units (each MB = 16 pixels). Makes the ghost softer and more diffuse. At high values the ghost becomes a blurry wash rather than a crisp echo. Requires Ghost Blend > 0 and Ref Depth ≥ 1.
Spill Radius 0 – 8 MB Expands the painted MB selection outward by this many macroblocks in every direction before applying any effects. Effectively blooms the selected region. Widens the zone of effect without repainting. A single painted MB with spill=3 affects a 7×7 MB region. Use to create soft-edged corruption regions.

Knob Reference — NOISE section

Control Range What It Technically Does Visual Result
QP Delta −51 – +51 Injects a per-MB quantisation parameter offset via the encoder's ROI (Region of Interest) side data. Positive values force higher QP (coarser quantisation, larger DCT coefficient rounding, more information loss). QP=51 is the maximum — all DCT residuals round to zero. Positive QP Delta = more compression artefacts, blockier image, loss of fine detail in painted MBs. At +51 the encoder outputs skip macroblocks — it cannot add any correction residuals, so only the motion-compensated prediction survives. This is what locks in datamosh corruption. Negative QP Delta = higher quality than surrounding areas.
Noise Level 0 – 255 Adds random ±N luma (Y-plane) noise to every pixel in selected MBs before encoding. Grain, static, snow. At low values adds subtle film grain. At high values (150+) produces heavy analogue static. The encoder cannot efficiently encode random noise, so it also indirectly increases the apparent QP in those MBs.
Pixel Offset −127 – +127 Adds a fixed DC offset to all luma pixel values in selected MBs (clamped to 0–255). Brightens (positive) or darkens (negative) painted MBs. Combined with QP=51 creates overexposed white patches or underexposed black patches that the encoder can't correct.
Invert Luma % 0 – 100 % Blends luma pixels toward their inverse (255 − value). 100% = full inversion (photographic negative of brightness). Photographic negative effect on painted MBs. Partial inversion produces a grey washed-out look. Confuses the encoder's rate-distortion decisions, often producing artefacts at MB borders.

Knob Reference — CHROMA section

Control Range What It Technically Does Visual Result
Chroma Drift X −127 – +127 px Shifts the U and V (chroma) planes horizontally by this amount (in half-pixel units for 4:2:0), sampling from the reference frame. Luma is unaffected. Colour fringing / chroma aberration — colour bleeds to the left or right of edges while brightness stays in place. VHS colour bleed aesthetic. Requires Ref Depth ≥ 1.
Chroma Drift Y −127 – +127 px Same but vertical chroma shift. Vertical colour fringing. Combine with X for diagonal chroma bleed.
Chroma Offset −127 – +127 Adds a fixed DC offset to both U and V chroma planes. Shifts the overall hue of painted MBs. Positive = more green/magenta tint (YCbCr space). Combine with Twist U/V for fine hue control.
Twist U −127 – +127 Adds a DC offset to the U (Cb) chroma plane only. Steers hue toward blue (negative) or yellow (positive) independently of V. Use opposite values on U and V for strong hue rotations.
Twist V −127 – +127 Adds a DC offset to the V (Cr) chroma plane only. Steers hue toward cyan/green (negative) or red/magenta (positive).

Knob Reference — PIXEL MANIPULATION section

Control Range What It Technically Does Visual Result
Flatten % 0 – 100 % Blends each macroblock's luma pixels toward the spatial average (DC value) of that MB. 100% = every pixel in the MB becomes the same grey value (the average). Block averaging / posterisation. At high values the image resolves into flat colour tiles whose hue is the average of the original. The encoder sees perfectly uniform 16×16 blocks, forcing very coarse quantisation decisions and producing heavy DCT ringing at borders.
Scatter px 0 – 32 px For each MB, randomly samples a reference frame position within ±scatter pixels of the correct position. The sampled content is written into the current frame. Shattered-glass or cubist dissolve. Each MB gets content from a slightly different random location in the reference, breaking spatial coherence. Higher scatter = more fragmented look. Requires Ref Depth ≥ 1.

Transient Envelope — Cascade Controls

The cascade controls are at the bottom of the MB editor panel. They cause an effect on the current frame to automatically propagate forward in time across subsequent frames with optional decay.

Control Range What It Technically Does Visual Result
Cascade Length 0 – 60 frames When > 0, the encoder automatically generates descendant edit entries for the next N frames after the seed frame. Each descendant copies from the immediately-preceding (already-corrupted) frame via Ghost Blend, and applies QP=51 to prevent the encoder from correcting the corruption. Only the current frame's cascade length is set — this control does NOT range-propagate. The corruption from the current frame "survives" for N more frames. The datamosh smear persists instead of healing to clean content after one frame. Set to 0 to corrupt only the current frame. Set high for a long sustained smear.
Cascade Decay 0 – 100 % Controls how much the effect intensity fades over the cascade. Uses a linear fade: at 0% the effect remains at full intensity for all N frames. At 100% the effect fades linearly from full intensity to zero by the final cascade frame. At 50% it fades from 100% to 50% over the cascade length. 0% = flat sustained smear (think "Full Freeze"). 50% = gentle fade that still produces noticeable corruption at the end. 100% = the smear dissolves completely by the last cascade frame, giving a natural-looking decay. Only the current frame's cascade decay is set.
Cascade Self-Blocking — Important
If you set Cascade Length on every frame (e.g. by painting all frames and using the slider), every frame becomes a "seed" and the cascade guard prevents any descendant from being generated (it sees every future frame already occupied). For cascades to work, only seed frames should have a non-zero Cascade Length. Descendant frames must be left at zero.

§7 — Global Encode Parameters

These settings control how x264 (the underlying H.264 encoder) makes decisions during every encode pass. They do not add pixel manipulation — they shape the encoder's rate control, motion estimation, frame structure, and quantisation. Press Apply Global Params + Re-encode to apply them without any MB edits.

Apply Global Params does NOT apply MB edits
This button re-encodes the video using only the encoder settings. No per-frame pixel manipulation is performed. Use it to adjust codec behaviour (e.g. switch to infinite GOP, change deblocking) without adding new corruption.

Kill I-Frames (top checkbox)

The Most Important Control in the Application

When checked, every I-frame in the source (except frame 0) is forced to a P-frame during re-encode. The encoder uses inter-prediction everywhere, maintaining a continuous chain of motion-compensated frames. This is the prerequisite for datamoshing.

When Kill I-Frames is active it also forces GOP Size to ∞ (keyint=9999) regardless of your GOP Size setting. Without this, x264 would insert a mandatory I-frame at every GOP boundary even if all frames are hinted as P, resetting the datamosh chain at regular intervals.

B-frames are also disabled when Kill I-Frames is active (bframes=0, b-adapt=0). x264's B-frame lookahead can insert I-frames to anchor the B-frame reference chain, defeating Kill I-Frames.

Frame Structure

Control Range What It Technically Does Visual / Datamosh Result
GOP Size −1 / 0 / 1–9999 Sets how often the encoder inserts a keyframe. −1 = encoder default (every 250 frames). 0 = infinite (no automatic keyframes ever). Positive = exact frame interval. Note: overridden to 9999 when Kill I-Frames is active. −1: I-frame every 250 frames — the chain resets periodically. 0 (infinite): no automatic resets — datamosh smear runs to the end of the clip. Set to 0 for any datamosh work.
B-Frames −1 – 16 Maximum number of consecutive B-frames. −1 = encoder default (3). Forced to 0 when Kill I-Frames is on. More B-frames → more temporal ghosting from past+future blending. 0 → pure P-chain, more stable datamosh. Higher values with Kill I-Frames off create layered temporal ghost effects.
B-Adapt Off / Fast / Trellis How x264 decides where to insert B-frames. Forced to Off when Kill I-Frames is active. Trellis = full rate-distortion optimal placement. Trellis placement produces the "smoothest" B-frame transitions. Off means B-frames are placed at fixed intervals.
Ref Frames −1 – 16 How many past frames the encoder can reference for motion compensation. −1 = default (3). More refs = encoder searches further back in time. Higher ref count → the encoder can copy content from further back in time → longer temporal smear tails. Set to 16 with SmearHeavy preset for maximum temporal bleed.

Rate Control

Control Range What It Technically Does Visual Result
QP Override −1 – 51 −1 = use CRF 18 (near-lossless rate control). 0–51 = switch to constant-QP mode, forcing every MB to this exact quantiser regardless of content complexity. −1 (CRF 18): high quality, variable bitrate. 51: maximum compression, extreme blocking artefacts, near-zero residuals everywhere. Values 30–45 produce visible but variable artefact levels.
QP Min −1 – 51 Floor on the quantiser — the encoder cannot use a QP lower than this. −1 = no limit. Setting a high QP Min forces a minimum corruption level even on easy-to-encode regions. All frames become at least this damaged.
QP Max −1 – 51 Ceiling on the quantiser. −1 = no limit. 51 = the encoder can use maximum quantisation but no higher. Setting QP Max to 51 with a high QP Min creates a "forced corruption" floor where the encoder must produce artefacts everywhere. The DataCorrupt preset sets QP Min=36, Max=51.

Motion Estimation

Control Range What It Technically Does Visual Result
ME Method Default / dia / hex / umh / esa / tesa The algorithm x264 uses to search for the best motion vector for each MB. dia = diamond search (fast, inaccurate). esa/tesa = exhaustive search (slow, accurate). dia (inaccurate): the encoder picks wrong motion vectors → content from incorrect reference positions bleeds in → smear, jitter, drift. esa (accurate): the encoder correctly compensates for motion → clean result even with large pixel manipulations. Use dia for more datamosh artefacts.
ME Range 0 – 512 px How many pixels the encoder searches in each direction when looking for the best reference block. 0 = default. Narrow range → encoder gives up searching early → uses wrong MVs → more artefacts. Wide range → finds correct matches over larger distances → cleaner result. Use narrow range to amplify datamosh drift.
Subpel Ref −1 – 10 Sub-pixel refinement quality. 0 = integer-pixel MVs only. 10 = full quarter-pixel precision. −1 = default. 0: MVs snap to full-pixel boundaries → staircase artefacts at edges, more visible blocking. 10: smooth sub-pixel compensation → cleaner motion. Use 0 or 1 for gritty, harsh artefact textures.

Partitions & DCT

Control Range What It Technically Does Visual Result
Partition Mode Default / 16×16 / p8×8 / All / All+4×4 Which MB sub-partition sizes the encoder tests when searching for motion vectors. 16×16 only = one MV per MB. All+4×4 = up to 16 sub-block MVs per MB. 16×16 only: one crude motion vector per block → large blocky artefacts, mosaic look. All+4×4: very fine-grained motion → accurate but complex artefacts. The BlockMosaic preset uses 16×16 for the classic tile-glitch aesthetic.
8×8 DCT On / Off Enables 8×8 DCT transform blocks (High profile feature). On = encoder can use both 4×4 and 8×8 transform blocks. Off = 4×4 only. Off = lower-frequency DCT capture → slightly softer, more 4×4-block artefact character. On (default) = sharper frequency response.

Quantisation Flags

Control Range What It Technically Does Visual Result
Trellis Default / Off / Final MB / All MBs Rate-distortion optimal coefficient zeroing after DCT. Off = no trellis, noisy residual selection. All MBs = every MB's DCT coefficients are trellis-optimised. Off: noisier residuals, more random-looking coefficient patterns, less efficient encoding = different artefact character. All MBs: cleanest residuals, most efficient. Use Off for a more "raw" corrupt look.
No Fast P-Skip On / Off Forces the encoder to fully evaluate every MB's residual even when early-termination (fast skip) would normally bail out. Makes encoding slower but more thorough. Slightly different artefact distribution — residuals that would have been zeroed by fast skip are now evaluated and possibly retained. Subtle textural difference in corruption patterns.
No DCT Decimate On / Off Normally x264 discards near-zero DCT coefficients. Enabling this preserves them. Preserves micro-level frequency information that would normally be thrown away. Produces a finer, more granular noise texture — less blocky, more high-frequency artefact character. Used in QuantumResidue preset.
CAVLC (no CABAC) On / Off Switches the entropy coding stage from CABAC (context-adaptive binary arithmetic coding) to CAVLC (variable-length coding). CAVLC is less efficient. Slightly different coefficient distribution in the compressed stream → subtly different artefact character. Lower compression efficiency means more bits used → larger file.

Deblocking Filter

Control Range What It Technically Does Visual Result
Disable Loop Filter On / Off Turns off the H.264 in-loop deblocking filter that normally smooths 16×16 block boundaries in the reconstruction buffer. Off (disabled): raw 16×16 macroblock grid becomes visible as a grid of sharp edges. Produces the classic "blocky" compression artefact aesthetic. Essential for block-mosaic looks. All datamosh presets that want hard block edges use this.
Alpha α / Beta β −6 – +6 Tunes the deblocking filter strength when it is enabled. Alpha controls the edge threshold; Beta controls the flatness threshold. Negative = weaker filter (preserves more edges). Positive = stronger filter (smooths more aggressively). Negative values let more block boundary artefacts through even with the filter on. Positive values aggressively smooth even pronounced blocks. Fine-tune the "softness" of artefact edges.

Psychovisual Optimisation

Control Range What It Technically Does Visual Result
Psy-RD −1 – 5.0 Rate-distortion cost that penalises reduction of perceptual detail. −1 = default (1.0). 0 = off (flat energy distribution). High values = encoder adds grain and texture to maintain apparent detail. 0: clean, flat, minimal residuals — less "textured" artefacts. High (3+): encoder adds synthetic grain to compensate for information lost to quantisation. Combine with high QP for gritty, organic-looking corruption.
Psy-Trellis −1 – 1.0 Psychovisual trellis optimisation strength. Adds perceptual bias to the trellis coefficient selection. Higher values produce slightly more "alive" textural artefacts at the cost of strict rate control.
AQ Mode Default / Off / Variance / Auto-Var / Auto+Edge Adaptive quantisation — redistributes bits within a frame based on spatial complexity. Off = flat QP across the whole frame. Auto+Edge = prioritises edges and high-detail regions. Off: flat QP everywhere, more artefacts in complex regions. Auto+Edge: allocates more quality to edges, which can interfere with intentional block-edge artefacts. For maximum datamosh destruction, set AQ Off.
AQ Strength −1 – 3.0 How aggressively AQ redistributes QP within the frame. −1 = default. Higher = more extreme per-region QP variation. High AQ strength with ChromaFever preset creates dramatic per-region quality swings, producing a psychedelic chroma saturation bleed in areas of high spatial variance.
Disable MB-Tree RC On / Off MB-Tree is a lookahead rate control that pre-analyses frame propagation to assign more bits to MBs that will be referenced frequently. Disabling removes this intelligence. Disabled = more uniform bit allocation regardless of temporal importance. MBs that are rarely referenced get the same treatment as ones referenced many times — less "smart" encoding = different artefact distribution.
Lookahead −1 – 250 frames How many frames ahead the encoder looks before deciding quantisation for the current frame. −1 = default. 0 = no lookahead. 0 = truly blind encoding — the encoder cannot anticipate scene changes or reference patterns. Combined with datamosh parameters, short lookahead creates more unpredictable artefacts.

Spatial Mask

The Spatial Mask lets you nominate a region (captured from the MB painter canvas) that gets a persistent QP override applied to every single frame of the encode — not just edited frames.

  • Paint MBs in the MB editor canvas
  • Click Capture Current MB Selection in the Global Params panel
  • Set Mask QP (1–51; 51 = force skip MBs in that region on every frame)
  • Press any Apply button — the mask is applied globally throughout the encode
  • Click Clear Mask to remove it
Spatial Mask Use Case
Paint a rectangle around the subject's face. Capture it. Set Mask QP=51. Now every encode will maximally corrupt that region on every frame regardless of other MB edits, while the rest of the frame remains at normal quality.

§9 — Control Relationships & Dependencies

Many controls only have an effect when certain other controls are also set correctly. Here is a complete map of interdependencies.

Reference Chain Dependencies

The following MB Editor knobs require Ref Depth ≥ 1 to have any effect. If Ref Depth is 0, these controls are silently ignored:

  • MV Drift X / Y — needs a reference frame to shift pixels from
  • MV Amplify — multiplies Drift X/Y; meaningless if Drift is 0 or ref=0
  • Ghost Blend % — blends toward reference; needs ref=0 to have a source
  • Sample Radius — blurs the ghost blend source; requires Ghost Blend > 0 and Ref Depth ≥ 1
  • Chroma Drift X / Y — shifts chroma from reference; requires Ref Depth ≥ 1
  • Ref Scatter — samples from random offsets in reference; requires Ref Depth ≥ 1
Set Ref Depth = 1 for Any Temporal Effect
This is the single most common misconfiguration. If your MV drift, ghost blend, or chroma drift seem to do nothing, check that Ref Depth is set to at least 1.

Kill I-Frames Cascade

When Kill I-Frames is checked, the following are automatically overridden, regardless of what you have set:

  • GOP Size → 9999 (∞) — prevents x264's keyint schedule from inserting I-frames at regular intervals
  • B-Frames → 0 — prevents x264's B-frame lookahead from inserting I-frames to anchor B-reference chains
  • B-Adapt → Off — same reason
  • AV_FRAME_FLAG_KEY cleared on every non-IDR frame — prevents decoded I-frame key flags from triggering IDR encoding even with pict_type=P set

QP Delta vs QP Override

  • QP Delta (MB Editor) — per-MB relative offset applied via ROI side data. It is added to the frame's base QP. A QP Delta of +30 on a frame with base QP=18 → effective QP ≈ 48 on those MBs.
  • QP Override (Global) — forces a fixed absolute QP on all MBs across all frames, replacing CRF rate control entirely. QP Override = 51 → QP=51 everywhere, no rate control.
  • Both can be active simultaneously: QP Override sets the base, QP Delta adds on top (clamped to 51).

Cascade & Propagation

  • Cascade Length (MB editor) only generates descendants for frames that do not already have explicit edits. Setting cascadeLen on frame 1 with cascadeLen=50 and all 50 following frames also having explicit edits means the cascade generates nothing — the guard blocks it.
  • Kill I-Frames must be on for cascade to be effective. Without it, x264 re-inserts I-frames that reset the cascade chain.
  • Cascade only writes to the current frame in the MB editor — it does not propagate to the active range selection. This is intentional.

Spill Radius vs Painted MBs

  • Spill Radius expands the working set of MBs beyond the painted set. The painted MBs are the seed; spill grows the zone of effect outward. A frame in Global Mode (no painted MBs) with Spill Radius set uses all MBs as the base — spill has no additional effect in global mode.

Quick Mosh vs MB Editor Presets

  • Quick Mosh presets — whole-video, always cascade from frame 1 or stamp all frames. Always force Kill I-Frames + infinite GOP.
  • MB Editor presets — apply to selected frames only (or all frames if all are selected). Use the active range. Do NOT force Kill I-Frames — you must check it yourself in Global Params if needed.

Spatial Mask vs MB Editor Edits

  • Both apply QP ROI side data. If a frame has both a MB editor QP Delta and a Spatial Mask active on the same MB, both ROI entries exist — the encoder applies both offsets. The result is additive QP pressure on that MB.

§10 — Recommended Workflows

Workflow A — Classic Datamosh (Delete-Based)

How This Works
Delete uses a bitstream splice — it physically removes compressed packets without re-encoding. The surviving P-frames keep their original H.264 motion vectors, which still reference the deleted content. The decoder applies those vectors to the wrong shot's content → smear. This only works on I/P-only streams because B-frames are packets out of display order, which would cause the wrong frames to be removed.
  1. Load a video that has two or more distinct shots (a hard cut works best).
  2. In the timeline, select all frames (click the first frame, then Shift+click the last).
  3. In the Global Params panel, check Kill I-Frames and set GOP Size to 0.
  4. Press Force → P. This re-encodes the video so every frame except frame 0 is a P-frame. The timeline should now show only one white (I) block at frame 0 and all blue thereafter.
  5. In the timeline, select the frames you want to remove — typically the frames at or just after a scene cut. These are the frames whose deletion will "break" the motion vector references of the frames that follow.
  6. Press ✖ Delete. The selected compressed packets are physically removed from the stream. The P-frames after the cut retain their original motion vectors, which now point to content from before the deleted section.
  7. Review in the Preview Player. The transition where frames were removed should display a smeary, blocky bleed-through of the previous shot's content. Repeat Delete on different ranges to compound the effect.
Global Params and MB Edits are not applied during Delete
Delete is a packet splice — no encoder is invoked. Kill I-Frames, QP, deblocking and all other encoder settings have no effect on the Delete step itself. Apply those settings via Force → P (step 4) before deleting.

Workflow B — Quick Mosh Then Refine

  1. Load a video.
  2. In Quick Mosh, select a preset (e.g. Vortex) and press Mosh Now!.
  3. Wait for the encode. The Preview Player auto-loads the result.
  4. The MB editor now contains the preset's edit map. Review it — navigate frames and inspect the knob values.
  5. Adjust specific frames: navigate to a frame, change knobs, press Apply MB Edits. This re-encodes on top of the Quick Mosh result.
  6. Try stacking a second Quick Mosh preset on top for compound effects.

Workflow C — Precision Per-Region Corruption

  1. Load a video. Apply Infinite GOP preset via Global Params.
  2. In the MB editor, navigate to the frame you want to corrupt.
  3. Paint the specific macroblocks you want to affect (e.g. just the subject's face).
  4. Set QP Delta = 51 and Ghost Blend = 80% and Ref Depth = 1.
  5. Set Cascade Length = 30 and Cascade Decay = 30%.
  6. Press Apply MB Edits.
  7. The painted region on that frame — and the next 30 frames — will ghost and freeze while the surrounding image remains clean.

Workflow D — Global Encoder Artefact Layering

  1. Apply a Quick Mosh preset to get a base layer of corruption.
  2. In Global Params, select the Max Datamosh or Block Mosaic preset.
  3. Press Apply Global Params + Re-encode.
  4. The already-corrupted file is now re-encoded with extreme encoder settings, amplifying and reinterpreting the existing artefacts through an additional lens of codec distortion.
  5. Repeat steps 2–4 with different presets to layer multiple artefact characters.
The Compounding Principle
LaMoshPit always reads the previously-encoded file as its input. Every Apply is a generational pass — corruption from previous passes is re-encoded through the new settings, often amplifying and transforming the effect unpredictably. Some of the most interesting results come from applying the same preset 3–5 times in a row, or from alternating between very different presets.

Control Debug Logging

The Enable Control Debug Logging checkbox (green, in Global Params) writes a detailed log of every knob change, MB paint stroke, and frame-parameter state to:

  • <app folder>/logs/LaMoshPit_ControlTest_Log.txt

Enable this when diagnosing unexpected results — the log shows exactly which parameters were sent to the encoder for each frame.
